
revision du XML :

1 - Définition :
    XML = eXtensible Markup Language
    XML est un langage de balisage utilisé pour stocker et transporter des données.
    Il permet de structurer l’information de manière lisible par l’homme et la machine.

2 - Structure générale d’un document XML :
    Un document XML est composé de :
        - Une déclaration XML (optionnelle mais recommandée)
        - Un seul élément racine

    Exemple :
        <?xml version="1.0" encoding="UTF-8"?>
        <racine>
            ...
        </racine>

3 - Les éléments XML :
    Un élément est défini par une balise ouvrante et une balise fermante.

    Exemple :
        <titre>Apprendre XML</titre>


4 - Les règles de base XML :
    - Un seul élément racine
    - Les balises doivent être fermées
    - Les balises doivent être correctement imbriquées
    - Les noms des balises sont sensibles à la casse
    - Les valeurs des attributs doivent être entre guillemets

5 - Les attributs :
    Les attributs permettent d’ajouter des informations à un élément.

    Syntaxe :
        <element attribut="valeur">

    Exemple :
        <livre id="l1" langue="fr">
            <titre>XML</titre>
        </livre>

6 - Le contenu des éléments :
    - Texte
    - Autres éléments
    - Les deux (contenu mixte)

7 - Les commentaires :
    Les commentaires ne sont pas affichés et ne font pas partie des données.

    Syntaxe :
        <!-- Ceci est un commentaire -->


8 - XML bien formé :
    Un document XML est bien formé s’il respecte toutes les règles syntaxiques XML.

    Exemple valide :
        <note>
            <to>Ali</to>
            <from>Aziz</from>
        </note>

    Exemple invalide :
        <note>
            <to>Ali</from>
        </note>

Résumé :
    - XML sert à représenter les données
    - Il impose des règles syntaxiques strictes
    - Il ne définit pas la structure (rôle du DTD / XSD)
